# Generator State Depth Recovery Design

Date: 2026-06-26
Status: Completed

## Problem

The maintained pure-Ruby generator increments a shared `JSON::State#depth`
while serializing arrays and hashes, but decrements it only after successful
completion. A circular or over-nested value raises during recursion and leaves
the state at its maximum depth. Every later structured generation with that
state then fails immediately, even for shallow valid input.

The defect reproduces on the pinned Ruby 2.7 archive runtime for both arrays
and hashes. The current upstream `ruby/json` generator at commit
`168095ccb80b21c2a1db2c6d0b6bf5682ef64ad9` keeps generation depth in
per-invocation data rather than leaving failed depth on a reusable state.

## Constraints

- Preserve generated bytes and indentation for successful calls.
- Preserve the configured starting depth rather than assuming it is zero.
- Cover direct collection `to_json(state)` calls as well as `JSON.generate`.
- Keep the executable change inside the maintained pure-Ruby boundary.
- Do not claim native-extension or JRuby runtime behavior without those legacy
  runtimes.

## Options

### Reset depth in `State#generate`

This repairs `JSON.generate` but not direct `Array#to_json(state)` or
`Hash#to_json(state)` calls, which are part of the archived public surface.

### Reset depth to zero when the limit is reached

This loses a caller-supplied starting depth and makes nested custom `to_json`
calls mutate state owned by an outer generation.

### Decrement each entered collection in `ensure`

Each array or hash restores exactly the depth increment it owns, whether child
generation succeeds or raises. Successful output remains byte-for-byte
unchanged, direct calls are covered, and nonzero starting depths are preserved.

## Decision

Wrap the array and hash transformation bodies in `ensure` blocks that decrement
their one owned depth increment. Use the captured entered depth for child
indentation and one level less for the closing delimiter.

## Validation

- Add a red-first regression for failed array and hash generation.
- Assert depth returns to its configured starting value and the state can
  immediately generate a shallow value.
- Run the focused generator suite and the full digest-pinned container gate.
- Compile all 12 archived Java sources with the documented Java 8 boundary to
  confirm the unrelated archive surface remains intact.

The implemented design passed the 13-test focused generator suite and the full
digest-pinned Ruby 2.7 `make check` archive gate. Hosted Java 8 source
compilation remains the exact-head proof for the unchanged JRuby source
boundary.
